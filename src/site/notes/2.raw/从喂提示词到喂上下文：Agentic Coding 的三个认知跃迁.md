别急着换模型。真正的杠杆不在模型——在你给它的上下文。大多数人用 AI 写代码的方式，就像开着跑车在泥地里踩油门：引擎再强，路面不对，速度就是上不去。这篇文章的核心观点只有一个：[[3.wiki/上下文工程\|Agentic Coding]] 的瓶颈已经从「写代码」转移到了「喂上下文」，大多数开发者在优化错误的东西。

## 短对话比长对话快

这是 [[3.wiki/AI Agent 系统\|AI 协作]]中最大的反直觉：看起来信息量更大的长对话，实际效果远不如一系列短对话。

Claude Code 团队在内部实践中发现一个规律——上下文越长，AI 的输出越不稳定。长对话会累积大量噪声：被读取但无关的文件、走不通的探索路径、已经失败的尝试。这些噪声挤占了模型的注意力预算，拖慢响应，推高成本。短对话保持上下文纯净，模型表现始终维持在最佳水平。

更反直觉的是，十个短对话的总信息量，远超一个等长的长对话。因为每个短对话都从干净状态出发，只包含高信号信息，而长对话是信号和噪声的混合物——越积越浑浊。

把「开新对话」视为正常工作流程的一部分，而不是失败后的重试。每个对话只做一件事，把大功能拆成不超过 15 分钟的子任务，子任务之间通过引用结论、查看 git 状态来传递必要信息。当 AI 开始重复、遗忘目标、像醉了一样兜圈子——这些「上下文腐化」的症状说明你需要的不是更大的窗口，而是更短的对话。

## 让环境替你说话

大多数人把精力花在打磨提示词上，但 Anthropic 内部实践揭示了一个更高效的方向：**让 AI 在环境中直接学到该怎么做**。

具体怎么做？三个动作。

**文档邻近化**：把模块职责和设计决策放在代码旁边的 README 中。AI 浏览目录时能立即建立心智模型，大幅减少无效搜索。

**秒级反馈**：确保 AI 知道如何运行局部测试而非全量套件。一个 3 秒就能出结果的单元测试，比一段 500 字的提示词更能帮 AI 校准方向。反馈越快，迭代越高效。

**让 AI 看到后果**：别在提示词里反复强调「不要用 `--no-verify`」。直接把这条规则写进 Git Hooks——AI 试一次被拦截，下次就记住了。将对 AI 的指导嵌入到错误输出中，比写更长的提示词有效得多。

环境设计的本质是：与其告诉 AI「什么不能做」，不如让环境直接让错误选项不可行。

## 复利系统：从喂上下文到积累知识

前面两个原则解决的是单次对话的效率。但真正的生产力飞跃来自系统设计——让每一次交互都为未来投资。

修完一个 Bug，你有两个选择。选择 A：结束对话，下次遇到类似问题从头排查。选择 B：花 5 分钟把根因和修复模式记录到项目的 `AGENTS.md` 中，下次 AI 直接给出方案。第一次选择 B 需要 5 分钟额外投入，但第三次、第十次遇到同类问题时，节省的时间会像复利一样累积。

三个关键动作：

- **沉淀经验到 `AGENTS.md`**：不是写一份巨长的指令文档，而是持续把 Bug 修复记录、已知陷阱和代码审查模式沉淀进去。配置文件的每一行都会影响每一个会话、每一个任务的产出，精简优于详尽。
- **让 Bug 修复产生长期价值**：修完 Bug 后问自己——能否通过 lint 规则预防？需要补充测试防止回归？一个真正的修复，应该让同类问题再也不会发生。
- **把成功模式固化为 SOP**：将验证过的实现模式总结为可复用的工作流模板，AI 执行类似任务时直接遵循。

`AGENTS.md` 的精简原则值得单独说。研究显示模型大约能可靠遵循 150–200 条指令，而 Agent 的系统提示本身已包含约 50 条。你的配置文件只放对所有任务都普遍适用的内容。特定场景的信息（数据库设计规范、API 规范）放在独立文件中，配置文件里只放一行指向它的路径。这种 **渐进式披露** 策略确保只有相关信息进入上下文，避免不必要的干扰。

## 从提示词到系统

当你把视野从单次对话拉到整个系统，会发现还有两层杠杆被大多数人忽视。

**[[3.wiki/Claude Code\|Claude Code]] Skills** 把特定领域的程序性知识（架构设计 SOP、特定库的最佳实践）封装为按需加载的模块，既节省 Token，又让通才模型在特定场景下表现得像专家。

**[[3.wiki/MCP\|MCP]]** 通过标准化的协议，让 AI 能直接连接数据库、文件系统、API 等外部资源，将 AI 从「对话工具」升级为真正的协作[[3.wiki/AI Agent 系统\|智能体]]。

真正的杠杆不在提示词的措辞，而在系统和环境的设计。这是从「用 AI 聊天」到「用 AI 做工程」的关键跨越。

AI 协作是一门需要大量实验才能掌握的手艺。与其不断打磨单一个提示词，不如开始构建你的复利系统——从记录三条 AI 容易犯错的地方开始。
