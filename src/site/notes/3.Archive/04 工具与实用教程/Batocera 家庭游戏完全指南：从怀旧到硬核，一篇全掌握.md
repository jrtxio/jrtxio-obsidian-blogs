## 一、Batocera 是什么？为什么它适合你？

**Batocera 是一个开箱即用的复古游戏操作系统**。

- 基于 Linux
- 内置 RetroArch 模拟框架
- 支持 **80+ 经典游戏平台**
- 专为 **电视 + 手柄 + 客厅场景** 设计

它的特点非常明确：

✅ 插 U 盘即可启动，无需安装  
✅ 手柄就是全部输入设备  
✅ 游戏自动分类、显示封面  
✅ 双人即插即玩  
✅ 即时存档 + 金手指，新手友好  
✅ 中文界面一键切换

**它不要求你精通系统，只要求你按下「开始」。**

## 二、先搞懂三个关键概念（新手 & 开发者都该知道）

### 1. ROM = 游戏本体

- 从原版卡带 / 光盘中提取的镜像文件
- 常见格式：`.nes`、`.sfc`、`.gba`、`.chd`
- 放入对应目录（如 `/roms/nes/`），Batocera 会自动识别

记住一句话即可：  
**ROM 放对目录，系统会替你做剩下的事。**

### 2. Hotkey = 系统功能键（⚠️非常重要）

**Hotkey 不是固定某个键，而是一颗被指定为“系统功能键”的按钮。**

- **手柄默认：`Select / Back`**
- 键盘默认：`Right Shift`

几乎所有系统级操作，都需要遵循一个规则：

> **按住 Hotkey，再按其他键**

新手遇到的所有问题里，  
**“退不出游戏”“找不到菜单”占了 90%**，  
而根本原因，几乎都是 **没搞懂 Hotkey**。

### 3. 金手指（Cheats）= 让游戏适应你

- 无限生命 / 无限子弹 / 一击必杀
- 不需要修改 ROM
- 游戏运行中随时开关

这不是作弊，而是：

> **让 30 年前只为高手设计的游戏，  
> 能被今天的普通人、孩子、家人轻松体验。**

## 三、📌 只有手柄，如何完整使用 Batocera？（核心一节）

Batocera 的设计目标就是：  
**完全不需要键盘，也能完整使用系统。**

下面这张表，**建议你第一次用时直接对照着看**。

### 🎮 Batocera 手柄快捷键速查表（默认映射）

> 默认 Hotkey = **Select / Back**

|场景|操作|
|---|---|
|主界面确认|**A**|
|返回上一级|**B**|
|移动选择|方向键 / 左摇杆|
|打开系统菜单|**Start**|
|进入游戏|**A**|
|退出游戏返回主界面|**Hotkey + Start**|
|打开游戏内菜单（RetroArch）|**Hotkey + X**|
|即时存档|**Hotkey + R**|
|即时读档|**Hotkey + L**|
|快进（部分核心支持）|**Hotkey + 右肩键**|
|重置当前游戏|**Hotkey + B**|
|关闭 RetroArch（安全退出）|菜单内选择 Quit|

> 💡 不同核心可能略有差异，但 **Hotkey + Start / Hotkey + X 是通用且最重要的**。

### 1️⃣ 主界面操作（像主机一样）

- **方向键 / 左摇杆**：移动
- **A**：确认
- **B**：返回
- **Start**：系统菜单（设置 / 关机）
- **Select**：Hotkey

你可以把 Batocera 理解成：  
**一台“没有鼠标、没有桌面、只有手柄”的主机。**

### 2️⃣ 进入 / 退出游戏（最重要）

- **A**：进入游戏
- **Hotkey + Start**：立刻退出游戏，返回主界面

👉 如果你只记住一个组合键，就记这个。

### 3️⃣ 游戏内操作（存档 / 金手指 / 设置）

在游戏运行时：

- **Hotkey + X**：打开 RetroArch 菜单

你可以在这里：

- Save State / Load State（即时存档）
- Cheats（金手指）
- Core Options（模拟器设置）
- Quit RetroArch（安全退出）

### 4️⃣ 即时存档：不再从头打

- **Hotkey + R**：存档
- **Hotkey + L**：读档

这是 Batocera **最适合家庭使用**的功能之一。

### 5️⃣ 完全不用键盘，可以吗？

**可以，而且这是推荐用法。**

键盘只在以下情况才需要：

- 第一次配置手柄
- 手动编辑配置文件
- 高级调试 / 开发用途

普通玩家 → **全程手柄即可**。

## 四、你的老电脑能跑哪些游戏？（性能指南）

> ⚠️ 模拟器是否流畅，  
> 和“游戏年代”关系不大，  
> 而取决于 **原主机硬件的复杂度**。

### 🖥️ 按硬件配置推荐（x86 电脑）

|设备|推荐平台|可尝试|避免|
|---|---|---|---|
|老旧笔记本（Celeron / Atom / 4GB）|NES、GB/GBC、街机（FBNeo）|SNES、GBA|PS1、N64、PSP|
|i3 / i5 4–7 代|16 位及以下全部|PS1、N64、DC|PS2、3DS|
|i5 8 代+ / Ryzen|几乎全部|PS2、3DS|PS3 / X360|

## 五、为什么《合金弹头》流畅，《最终幻想7》却卡？

不是因为 2D / 3D，而是复杂度：

- PS1 需要真实 **BIOS**
- 有大量 **FMV 视频**
- CD-ROM 流式读取
- 多硬件同步模拟

而《合金弹头》：

- 纯 2D
- 无视频
- 帧结构简单
- 对模拟器极其友好

👉 **不是 3D 卡，而是复杂才卡。**

## 六、性能优化建议（实用）

### 通用优化

1. PS1 / PSP 使用 **CHD 格式**
2. 关闭主界面 Smooth 滤镜
3. 音频采样率降到 32000 Hz

### 低配设备

- SNES → `Snes9x`
- PS1 → `PCSX-ReARMed`
- 避免高精度核心（如 bsnes）

## 七、技术深潜：模拟器到底在干什么？

- 模拟 ≠ 播放
- 模拟器在软件中重建整台硬件
- 精度越高，性能消耗越大
- JIT 极度依赖单核性能
- BIOS 是合法性与兼容性的门槛

这也是为什么：

> **有些“老游戏”，  
> 比你想象中更吃电脑。**
