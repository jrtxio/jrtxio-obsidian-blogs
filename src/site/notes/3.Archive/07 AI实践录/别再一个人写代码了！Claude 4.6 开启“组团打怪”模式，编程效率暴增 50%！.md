#ai

在 AI 编程领域，我们已经习惯了“一个指令，一个响应”的单兵作战模式。虽然 Subagent（子智能体）能帮我们分担不少脏活累活，但在面对复杂的系统重构、前后端并行开发时，单个 AI 的上下文瓶颈和串行逻辑依然让人捉襟见肘。

**直到 Claude Code Agent Teams 的出现。**

这不是简单的功能更新，而是 AI 编程从“单机时代”向“多人联机”的范式转移。今天，我们就来拆解这个让无数开发者高呼“离谱”的**多智能体协作系统**。

## 🚀 什么是 Agent Teams？

简单来说，**Agent Teams = 一个项目经理（Lead）+ 多个资深开发（Teammate）**,。

以前你用 Claude Code，它像是一个超级助理，虽然能干但只能一件件来。现在，你可以直接组建一支 AI 开发纵队：

- **Team Lead（负责人）**：负责需求拆解、任务分配和质量把控,。
- **Teammates（队友）**：独立的 Claude 实例，每个都有 **200K token 的独立上下文**，互不干扰,。
- **Shared Task List（共享看板）**：AI 界的 Jira，任务状态、依赖关系一目了然,。
- **Mailbox（消息系统）**：AI 之间的 Slack，支持点对点通信和全员广播,,。

**它与 Subagent 的核心区别是什么？** Subagent 是“下级向上级汇报”的星型结构，而 Agent Teams 是“队员之间直接沟通”的网状协作模式,,。如果你的任务需要讨论、质疑和高度并行，Agent Teams 是唯一选择,。

## 🛠️ 手把手教你组建“AI 开发部”

#### 第一步：开启隐藏开关

Agent Teams 目前还是实验性功能，默认关闭。你需要在 `settings.json` 中添加环境变量：

```
{ "env": { "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1" } }
```

或者在启动时指定：`export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`,,。

#### 第二步：自然语言召唤团队

不需要复杂的斜杠命令，直接对 Claude 下令：

> “帮我重构认证模块，创建一个包含 3 个队友的团队：一个负责重构登录，一个负责注册，一个负责写测试，统一使用 Sonnet 模型。”,

#### 第三步：开启“监控墙”模式（推荐）

如果你想实时看 AI 们是怎么“吵架”和敲代码的，强烈建议开启 **tmux 分屏模式**,,。 在 `settings.json` 中设置 `"teammateMode": "tmux"`，你就能看到每个队友在独立的终端窗口中疯狂输出，这种上帝视角简直是 Vibe Coding 的极致享受,,。

## 🔥 实战场景：1 小时写完一个宝可梦 RPG？

有开发者测试，用 Agent Teams 开发一个包含战斗系统、地图探索、存档功能的宝可梦风格游戏，仅需 **1-2 小时**,。

- **Teammate A** 负责战斗核心逻辑；
- **Teammate B** 负责 UI 与动画；
- **Teammate C** 负责地图渲染；
- **Lead Agent** 负责定义接口协议（Contract）并整合代码。

**对比单 Prompt 模式**：普通 AI 会因为上下文太挤而“偷懒”省略细节，但 Teams 模式下每个队员各司其职，最终产出的代码量可达 **2000+ 行**，且逻辑更完整、更专业,。

## 💡 避坑指南：如何用得更省、更稳？

1. **混合模型搭配（省钱秘籍）**：让最聪明的 **Opus 4.6** 当 Lead 负责战略决策，让性价比最高的 **Sonnet** 当 Teammate 负责写具体代码。这样既能保证项目不跑偏，又能控制 Token 消耗,。
2. **避免文件冲突**：两个 AI 同时改一个文件会发生覆盖，这是目前最大的痛点,,。**最佳实践是：一人负责一个目录或一组独立文件**,,。
3. **合同优先（Contract-First）**：在并行开发前，先让 Lead 定义好 API 接口和数据结构。否则队员 A 写的后端和队员 B 写的前端拼在一起时，可能会因为字段名对不上而“翻车”,。
4. **善用 Shift+Tab（委派模式）**：如果你发现 Lead 老是忍不住自己下场写代码，按 `Shift+Tab` 开启委派模式，强迫它只管调度，不许动手,,。

## ⚠️ 注意事项与成本

- **Token 碎钞机**：3 人团队的 Token 消耗约为单人的 **3-4 倍**,,。对于简单修改，单实例更划算,。
- **实验性限制**：目前不支持会话恢复（/resume），一旦关闭，队员的进度就找不回来了,,。
- **读密集型是甜点区**：Agent Teams 最擅长的是**多角度代码审查**（一个查安全、一个查性能、一个查逻辑）和**竞争假设调试**,,,。
